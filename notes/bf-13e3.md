# NOAA Archive URLs for Lambert-Conformal GRIB2 Files

## Summary

Research completed for finding publicly accessible NOAA archive URLs hosting Lambert-conformal (GDT 3.30) GRIB2 files.

## Key Finding: NAM CONUS Nest Uses Lambert Conformal Projection

**Model:** NOAA NAM (North American Mesoscale) CONUS Nest  
**Grid:** AWIPS Grid 218 / CONUS Nest  
**Projection:** Lambert Conformal Conic (GDT 3.30)  
**Resolution:** 12 km (Grid 218) and 3 km (CONUS Nest hiresf)

## File Naming Convention

```
nam.tCCz.conusnest.hiresfFF.tm00.grib2
```

Where:
- `CC` = Cycle time (00, 06, 12, 18 - representing 0Z, 6Z, 12Z, 18Z model runs)
- `FF` = Forecast hour (00-84, typically at 3-hour intervals)
- `hiresf` = High-resolution CONUS nest files
- `tm00` = Time indicator

## Grid Definition Template (GDT) Clarification

**GDT 3.30** is the Grid Definition Template for **Lambert Conformal Conic** projection in GRIB2 files.

### NAM Grid 218 Specification (from NCEP documentation):
- Grid: AWIPS Grid 218
- Projection: Lambert Conformal Conic
- Resolution: 12 km
- Coverage: Continental United States (CONUS)

### Grid 227 (CONUS Nest):
- Grid: AWIPS Grid 227
- Projection: Lambert Conformal Conic
- Resolution: 3 km (high-resolution nest)
- Coverage: Continental United States (CONUS)

## Data Representation Template (DRT) Clarification

**Important Note:** The task description mentions "DRT=3" but this appears to be terminology confusion:

- **GDT 3.30** = Grid Definition Template (Section 3) - Lambert Conformal projection ✅
- **DRT** = Data Representation Template (Section 5) - No "DRT 3" in standard NCEP/WMO GRIB2 documentation

Standard Data Representation Templates for GRIB2:
- **DRT 5.0** = Grid Point Data - Simple Packing
- **DRT 5.2** = Grid Point Data - Complex Packing
- **DRT 5.3** = Grid Point Data - Complex Packing + Spatial Differencing

NAM CONUS nest files typically use **complex packing** (DRT 5.2 or 5.3) for data compression.

## Additional Notes

- AWS S3 access is generally faster and more reliable than NOMADS
- Historical data availability varies: recent data (30-90 days) on NOMADS; longer archives on AWS S3
- Both sources provide the same GRIB2 files with identical internal structure
- Files are large (hundreds of MB to 1+ GB) - allow sufficient download time
