# Bead bf-5y18: Format Output to Match Golden JSON Structure

## Task Completed

Successfully aligned `gen_golden.py` output with expected golden format.

## Problem Analysis

The original `gen_golden.py` script was generating JSON in a different format than the expected golden file structure:

### Original Format Issues
- **parameter** field included extra subfields: `short_name`, `long_name`, `units`, `table_source`
- **forecast** field had nested `reference_time` object with `iso_string`, `significance_name`, `components`
- **level** field had complex nested structure with `first_level` and `second_level` objects
- Missing required fields: `drt_template`, `packing`, `values`
- **grid** field had different structure with `ni`, `nj` instead of `nx`, `ny`, and `grid_type` field

### Expected Golden Format
- **parameter**: `{discipline, category, number}` (three integers only)
- **forecast**: `{reference_time: {year, month, day, hour, minute, second, significance}, time_range_unit, forecast_offset}`
- **level**: `{type1, scale_factor1, scaled_value1, type2, scale_factor2, scaled_value2}` (flat structure)
- **ensemble**: `null` or `{member_type, number}`
- **grid**: `{template, num_data_points, nx, ny, lat_first, lon_first, lat_last, lon_last, di, dj, scanning_mode, resolution_flags, shape_of_earth}`
- **values**: `{Dense: [...]}` or `null`
- **packing**: `{reference_value, binary_scale_factor, decimal_scale_factor, bits_per_value, original_field_type}`
- **drt_template**: integer

## Solution

Replaced `scripts/gen_golden.py` with `scripts/gen_golden_fixed.py`, which already implements the correct golden format structure.
