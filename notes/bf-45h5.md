## Verification Method

### Tool Used
- **Python eccodes library** (ECMWF GRIB API version 2.46.0)
- Alternative to wgrib2 (wgrib2 not available via apt, compilation failed on Debian Trixie)
- Eccodes provides equivalent functionality for inspecting GRIB2 templates

### Inspection Process
```python
import eccodes

with open('file.grib2', 'rb') as f:
    while True:
        msg_id = eccodes.codes_grib_new_from_file(f)
        if msg_id is None:
            break

        gdt = eccodes.codes_get(msg_id, 'gridDefinitionTemplateNumber')
        drt = eccodes.codes_get(msg_id, 'dataRepresentationTemplateNumber')
        grid_type = eccodes.codes_get(msg_id, 'gridType')
        # ... additional parameter inspection

        eccodes.codes_release(msg_id)
```

### Key Parameters Inspected
- `gridDefinitionTemplateNumber`: 30 = GDT 3.30 (Lambert Conformal Conic)
- `dataRepresentationTemplateNumber`: 3 = DRT=3 (Complex packing with spatial differencing)
- `gridType`: Projection type (e.g., "lambert")
- `Nx`, `Ny`: Grid dimensions
- `paramId`: GRIB parameter code
- `name`: Parameter name

## Technical Notes

### GDT 3.30 (Lambert Conformal Conic)
- Grid Definition Template number 30 corresponds to GDT 3.30
- Used for regional models with Lambert Conformal projection
- Standard for CONUS regional weather models
- Provides better representation of regional features than regular lat/lon grids

### DRT=3 (Complex Packing)
- Data Representation Template number 3
- Complex packing with spatial differencing
- Provides significantly better compression than simple packing (DRT=0)
- Standard for high-resolution GRIB2 files
- Some special fields (categorical data) may use simple packing (DRT=0)

### Tool Selection
- **wgrib2:** Not available via apt package manager, compilation failed due to missing dependencies on Debian Trixie
- **eccodes:** Python library from ECMWF, provides equivalent functionality, successfully used for verification
- Both tools provide access to GRIB2 template numbers and metadata
