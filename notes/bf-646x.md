## Grid Definition Template (GDT) Format

The GDT information is presented in the output as:

```json
"grid": {
  "template": 0,              // GDT number
  "num_data_points": 25,      // Total grid points
  "nx": 5,                     // X dimension
  "ny": 5,                     // Y dimension
  "lat_first": 40,             // First latitude point
  "lon_first": 0,              // First longitude point
  "lat_last": 0,               // Last latitude point
  "lon_last": 40,              // Last longitude point
  "di": 10,                    // Longitudinal grid spacing
  "dj": 10,                    // Latitudinal grid spacing
  "scanning_mode": 0,          // Scan mode flags
  "resolution_flags": 48,      // Resolution flags
  "shape_of_earth": 6,         // Earth shape specification
  "projection": "LatLon"       // Grid projection type
},
"templates": {
  "gdt": 0,                    // Grid Definition Template number
  "pdt": 0,                    // Product Definition Template number
  "drt": 0                     // Data Representation Template number
}
```

## GDT Templates Found

All successfully decoded files in this inspection used **GDT 0** (Regular Latitude-Longitude Grid). This is the most common grid template for global and regional gridded data.

**GDT 0 Characteristics:**
- Regularly spaced latitude-longitude grid
- Defined by lat/lon extents and grid spacing (di, dj)
- Simple rectangular grid structure

## Notes
- Some files with newer or more complex grid templates (like rotated lat-lon) are not yet implemented in gribtract
- The tool successfully decoded files with DRT 0, 2, and 3 (different packing methods)
- All template information (GDT, PDT, DRT) is clearly identified in the JSON output

## Command Used
```bash
cargo run -p gribtract-cli -- list <GRIB2_FILE>
```

This provides comprehensive grid and template information equivalent to wgrib2 grid inspection functionality.
