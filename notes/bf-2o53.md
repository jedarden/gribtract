# Real Fixture Acquisition Plan for bf-2o53

## Goal
Expand differential corpus with real remote-stored fixtures from NOAA archives.

## Current State
- 9 fixtures in manifest: 6 synthetic, 3 real
- 0 fixtures use storage=remote (all inline or deferred)
- Missing coverage:
  1. Lambert-conformal (GDT 3.30) + DRT=3 complex packing
  2. Ensemble/statistical product (PDT 4.1/4.8)
  3. Gaussian-grid (GDT 3.40)
  4. CONUS-covering DRT=0 simple packing
  5. Full model cycle (40+ forecast hours)

## Acquisition Strategy

### 1. NAM Lambert DRT=3 (Priority 1)
**Source**: NOAA NAM 12km CONUS on AWS
- Archive: https://noaa-nam-pds.s3.amazonaws.com/index.html
- File pattern: `nam.YYYYMMDD/nam.tHHz.awphys12XY.tm00.grib2`
- Expected: GDT 3.30 (Lambert Conformal), DRT=3 complex packing
- Coverage: CONUS 12km resolution
- Target date: Recent cycle (2026-06-20 or later)
- Target variable: Temperature or precipitation
- Storage: remote (large file, ~50-100MB per field)

### 2. GEFS Ensemble PDT=4.1 (Priority 2)
**Source**: NOAA GEFS v12 on AWS
- Archive: https://noaa-gefs-pds.s3.amazonaws.com/index.html
- File pattern: `gefs.YYYYMMDD/HH/atmos/gep01.tHHz.pgrb2.0p25.fFFF`
- Expected: PDT 4.1 (individual ensemble member), global 0.25°
- Target: Member 1 (control), forecast hour f000
- Storage: remote

### 3. Gaussian Grid GDT 3.40 (Priority 3)
**Source**: NOAA GFS global Gaussian grid
- Archive: https://noaa-gfs-bdp-pds.s3.amazonaws.com/index.html
- File pattern: GFS on reduced Gaussian grid (if available) or extract specific field
- Alternative: ECMWF IFS (uses Gaussian grids) if NOAA lacks GDT 3.40
- Expected: GDT 3.40 (reduced Gaussian)
- Storage: remote

### 4. CONUS DRT=0 Simple Packing (Priority 4)
**Source**: NOAA GFS CONUS subset or legacy file
- Challenge: Modern GFS uses DRT=3; DRT=0 may require legacy archive
- Alternative: Extract DRT=0 field from multi-message GFS file
- Target: Any analysis field covering CONUS with simple packing
- Storage: inline (expected to be small)

### 5. Full GFS Cycle (Priority 5)
**Source**: NOAA GFS 0.25 degree global
- Archive: https://noaa-gfs-bdp-pds.s3.amazonaws.com/gfs.YYYYMMDD/HH/
- Files: `gfs.tHHz.pgrb2.0p25.f000` through `gfs.tHHz.pgrb2.0p25.f120`
- Expected: 40+ forecast hours for one field (e.g., TMP 2m)
- Storage: remote (very large, ~2GB for full cycle)
- Note: May be staged as follow-up due to size

## Implementation Plan

### Phase 1: Download and Verify
1. Download sample files from each source to /tmp/
2. Inspect with eccodes to verify GDT/PDT/DRT
3. Extract single-field subsets if needed
4. Compute SHA-256 and record metadata

### Phase 2: Store and Generate Goldens
1. Move verified files to tests/corpus/large/
2. Run `python3 scripts/gen_golden.py <file> <fixture_id>`
3. Store golden files in tests/corpus/golden/

### Phase 3: Update Manifest
1. Add manifest entries with:
   - `id`: descriptive fixture ID
   - `path`: large/<filename>.grib2
   - `sha256`: computed hash
   - `size_bytes`: file size
   - `storage`: "remote"
   - `provenance`: source URL and description
2. For inline fixtures (small DRT=0), use storage=inline

### Phase 4: Test Integration
1. Run differential suite: `cargo test differential`
2. Verify 100% agreement or create ratcheted fixtures
3. Run station benchmark to verify CONUS coverage
4. Update documentation as needed
